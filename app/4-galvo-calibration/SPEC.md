- 4. ガルバノスキャナのキャリブレーションページ
  - TypeScript の UI と C++ wasm、Web Serial API で実装
  - パス：/4-galvo-calibration
  - キャリブレーションは Web カメラを起点にして行う
  - UI はファイルシステムに似た remap ファイルを選択するシングルセレクト UI とシリアルデバイス選択 UI、実行ボタンが配置される。ユーザがファイルとシリアルデバイスを選択して実行ボタンを押すと以下のような流れで処理が走る
    - 専用の canvas に 3 のページ同様に変換された Web カメラの画像がリアルタイムに描画され、そのうちの 1 フレームがファイルシステムに「4-galvo-calibration/日時 + screen」という名前で保存される。ここの日時は以降の処理でも同じものが使われる。
    - マイコンから「ガルバノ移動 → レーザー出力 ON → レーザー出力 OFF → ガルバノ移動 → …」という処理が 1 秒サイクルで繰り返される。ガルバノは 2 つの鏡の角度をそれぞれ 16 ビット整数で制御するため、ある平面に対して格子状にレーザーの出力位置がプロットされる。格子幅は決めうちで、9 ビット刻みとかで OK。「レーザー出力 ON → レーザー出力 OFF」の間の画像が 1 枚づつファイルシステムに「4-galvo-calibration/日時 + _x_（ガルバノ x 座標）_y_（ガルバの y 座標）」という名前で保存される。ここでの日時は直前と同じものが使用される。
    - 「4-galvo-calibration/日時 + screen」の画像と各「4-galvo-calibration/日時 + _x_（ガルバノ x 座標）_y_（ガルバの y 座標）」の画像との間のピクセルレベルの diff を取ることで、レーザーの照射位置を特定し、ガルバノ座標とカメラ上の座標の対応点一覧を取得する
  - 上記の結果をもとに opencv の findHomography y 関数で変換行列を取得。ファイルシステムに「4-galvo-calibration/日時-homography」のファイル名で保存される
